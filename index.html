## Automated updates with Drupal
## Automated updates with Drupal - Was mentioned by Dries as part of the strategic roadmap of Drupal in his Driesnote at DrupalCon Vienna 2017 - There is a core initiative for it - Has been available in Wordpress for ages
## The Wordpress approach - Has a range of problems, among them _risk and security_ - Best suits hobbyists, for example with no version control - Not suited for projects with build steps or Continuous Integration
## Continuous integration >CI was intended to be used in combination with automated unit tests written through the practices of test-driven development.
## Continuous delivery >aims at building, testing, and releasing software with greater speed and frequency.

# Recap - Add Continuous integration - Add Continuous deployment - Add automated updates - Add automated PR merge - Profit
# Caveats - You have to have good test coverage - You might want to target security updates for full automation - If an update runs an update hook that creates or changes other files, it will not be fixed for you
